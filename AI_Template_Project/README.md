# AI 驱动标准项目脚手架 (AI-Driven Project Scaffold)

> **人类开发者 / AI 接入必读入口**

本项目的开发与维护已全面接入“基于哈希路由的 AI 自治”架构。任何人或 AI 在执行代码变更前，必须遵循本套件的规范。

## 🗺️ 架构地图与事实来源
- **入口**：请优先查阅 [`架构地图.md`](file:///架构地图.md) 及其包含的 `0.5 AI 任务触发字典`。
- **操作规范**：请遵循根目录的 `AGENTS.md`，严禁盲查代码，严禁绕过地图执行未记录的操作。

## ⚠️ 核心安全守则
1. **凭据隔离**：严禁在 `ops_scripts/` 或源码中硬编码密码或 Token。必须通过环境变量或 `config/credentials.env` 注入。
2. **强制备份**：在执行任何高危重构或数据库变更前，AI 必须先生成快照并保存在 `backups/` 目录中。
3. **经验下沉**：所有的特殊处理、Bug 修复，必须根据《地图写作规则》记录在 `maps/knowledge/` 目录下。

## 🛠️ 脚本命名规范 (`ops_scripts/`)
为了让 AI 能够一眼看出脚本用途，本目录下的脚本强制实行以下前缀命名法：
- `check_*.py/sh`：只读状态检查，无副作用。
- `deploy_*.py/sh`：服务部署与重启。
- `fix_*.py/sh`：一次性故障修补。
- `sync_*.py/sh`：数据或地图同步。
- `test_*.py/sh`：自动化验证。
