# AI 任务操作闭环 (Task Loop)

> 任何在这个项目中执行的代码修改，都必须严格遵守以下标准流程。

1. **意图分类**：明确用户是要“修Bug”、“加新功能”、还是“重构代码”。
2. **路由解析**：查阅 `架构地图.md`，锁定目标文件路径。
3. **直觉读取**：如果有复杂逻辑或诡异报错，去 `ai_intuition/` 里看看有没有前人的经验。
4. **执行修改**：按照指定的路径修改代码。
5. **验证结果**：说明你打算如何测试这个修改（哪怕是请求用户手动点击测试）。
6. **记录经验**：如果本次排查花费了较大精力，或者踩了坑，必须在 `maps/knowledge/经验库.md` 中记录一条。
