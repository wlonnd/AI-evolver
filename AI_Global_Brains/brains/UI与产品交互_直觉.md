# UI与产品交互直觉 (Brains)

> 本文件汇集了产品前端、交互设计以及页面状态流转层面的系统直觉。

## 1. 多设备状态同步与防陈旧
- **直觉**：长时间开着的静态网页或 SPA 页面（如后台看板、打卡页），极易在另一台设备发生操作后产生本地状态陈旧，进而引发用户困惑。
- **应对**：必须利用网页的可见性变化（`visibilitychange`、`focus`）或引入后台心跳定时器。当用户重新聚焦页面时，自动从服务器拉取并回流新状态，实现多端无缝切换。

## 2. 移动端防遮挡与跨端自适应
- **直觉**：在自定义复杂的 Web UI 时，底部或右下角的浮动控制栏极易挤占移动端 HTML5 `<video>` 的原生控件区（如全屏、音量按钮），导致核心功能报废。
- **应对**：在窄屏设备上，必须主动利用 CSS Media Query 变更布局（如将横排按钮拆为多行、或提高 `bottom` 定位偏移），让出系统级控件区域。绝对避免使用强制的定高或死板铺满，保留容器的自适应弹性。

## 3. 临时动作结果的持久化
- **直觉**：重度依赖临时跳转页、短暂 Toast 提示的交互极易让用户迷失。如果在跳转或提示闪过的瞬间用户没看清，他们将无法判断刚才的异步动作（如导入、提交表单）到底成功与否。
- **应对**：任何不可靠的异步变更动作，必须在页面主看板上留存一个“最近一次操作结果”的持久化状态块，使得用户在未看清临时反馈时仍能回看结论。

## 4. 管理页面的状态强压缩
- **直觉**：对于面向日常、低频维护用户的管理页面，如果仅仅透传底层的原始状态码（如 HTTP 403, 502, Cookie Invalid），反而会成倍增加小白用户的认知与解释负担。
- **应对**：不要让用户自己解释状态集合！必须在前端顶层将底层状态码再压缩、降维成一句直白的“可执行结论”，例如直接显示：“正常，无需更换 Cookie” 或 “凭证过期，请重新导入”。

## 5. 高危API与跨域限制的防降级意识
- **直觉**：凡是依赖特定“安全上下文”的浏览器原生 API（如 Clipboard 剪贴板读写 `navigator.clipboard`），或涉及跨站跨域（如 HTTPS 页面通过 AJAX 访问局域网 HTTP），必须在写代码的第一天就预设：**它一定会被浏览器拦截或静默失败**。
- **应对**：必须提供可靠的降级（Fallback）和退避手段。例如：剪贴板写入失败的 `.catch()` 中自动弹出版 `prompt` 供用户手动全选复制；受阻的页面内 Fetch 降级为通过顶层 `<form>` 发起的跳转式 Submit，从而绕开严格的 Mixed Content 策略。

## 6. 性能与复杂度的克制 (防渲染雪崩)
- **直觉**：在包含大量长文本或数学公式的高频交互页面中，“全盘 DOM 销毁重建”是绝对的性能毒药。同时，叠加过多极其消耗 GPU 的特效（如大面积的玻璃磨砂 `backdrop-filter: blur` 加上辉光阴影），会在低端设备或远程桌面上造成严重的延迟与雪崩。
- **应对**：必须克制！使用纯色（Solid）或极简描边代替高耗能特效。更新列表时务必使用“局部按需移除 (`remove()`) 或就地更新 DOM”，绝不全盘重排。对于重型第三方库（如 MathJax 排版），必须加入节流锁（Throttler），限制短时间内的触发频次。
- **布局约束**：对于特定展示框，要慎用死板的 `max-height` 及内部 `overflow: auto`。切忌在一个页面内出现多重内部滚动条撕裂体验。应利用层级全屏（`fixed inset: 0`）与主体锁定（`body { overflow: hidden }`）吸收局部多余空间，配合 `flex-shrink: 0` 保证关键头部内容百分百完整铺开。
