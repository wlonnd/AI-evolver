# 全局知识归档索引与路由 (AI 跨项目大脑皮层)

> **核心原则：道术分离与懒加载**
> 本目录下的知识为了避免上下文爆炸，采用**大脑(Brains)**与**领域(Domains)**彻底隔离的设计。任何介入新项目的 AI，必须根据当前任务的上下文，按需读取对应的文件。

## 目录结构

### 1. `brains/` (大脑直觉区 - 道)
存放**跨语言、跨框架**的高维原则、架构思维和排障直觉。新项目启动时，应当优先将这些心法注入。
*   `brains/系统架构_直觉.md`：并发锁策略、防重逻辑、缓存一致性、容错机制等系统设计直觉。
*   `brains/排障与调试_直觉.md`：看到错误日志不要盲目重试，如何定位真实死锁，系统性排查思维等。
*   `brains/UI与产品交互_直觉.md`：响应式防遮挡、降噪设计、结果状态压缩持久化等产品级直觉。

### 2. `domains/` (领域手册区 - 术)
存放**绑定特定技术栈**的具体死坑记录、配置模板、API 黑魔法。
*   `domains/前端Web_排坑手册.md`：React/Vue生命周期坑、CSS兼容、MathJax渲染性能坑、HTTPS跨域限制。
*   `domains/运维网络_排坑手册.md`：代理层网络自愈、URLTest组状态清除、端口一致性等。
*(遇到全新技术栈，AI 可在 `domains/` 下细粒度新建文件，如 `domains/Docker_排坑.md`，绝不允许大杂烩。)*

## 安全机制 (严格的 Git 闭环)
为了防止高价值知识被意外覆盖或删除：
1. **追加合并原则**：任何 AI 往此目录蒸馏知识时，只允许向上追加 (Append) 或精炼合并 (Refine)，**绝对禁止**直接删除独立直觉，除非用户明确授权。
2. **强制版本快照**：完成任何修改后，AI 必须使用 Bash 命令行执行 `git add .` 和 `git commit -m "自动蒸馏: ..."`，留下回滚快照。
